# CRMEB UniApp 项目说明文档

## 项目简介

CRMEB UniApp 项目是基于 UniApp 框架开发的移动端应用，支持多端发布，包括微信小程序、H5、App（Android/iOS）等平台。该项目是 CRMEB 开源商城系统的移动端前端部分，提供了完整的商城购物、用户管理、活动参与等功能。

## 技术栈

- **框架**: UniApp
- **前端框架**: Vue.js 2.x
- **UI 组件库**: uView UI
- **状态管理**: Vuex
- **网络请求**: Axios
- **小程序开发**: 微信小程序
- **App 开发**: 原生 App (Android/iOS)
- **H5 开发**: 移动端网页

## 开发环境

### 开发工具
- **HBuilderX**: 推荐使用 HBuilderX 3.0+ 版本进行开发
- **Node.js**: v12.0+（用于安装依赖）
- **微信开发者工具**: 用于微信小程序开发和调试

### 依赖管理
- 安装依赖：`npm install`

### HBuilderX 操作
1. **运行到微信小程序**:

2. **运行到 H5**:
   - 点击 HBuilderX 工具栏的 "运行" -> "运行到浏览器" -> 选择浏览器

3. **运行到 App**:
   - 点击 HBuilderX 工具栏的 "运行" -> "运行到手机或模拟器" -> 选择设备

### 命令行操作
- **安装依赖**: `npm install`
- **构建 H5**: 使用 HBuilderX 发行功能
- **构建小程序**: 使用 HBuilderX 发行功能
- **构建 App**: 使用 HBuilderX 发行功能

## 配置说明

### 应用配置
- **manifest.json**: 应用基本配置，包括应用名称、版本、权限等
- **pages.json**: 页面配置，包括页面路径、导航栏设置、底部标签栏等
- **config/app.js**: 应用业务配置，包括 API 地址、应用常量等

### API 配置
- **api/api.js**: API 基础配置，包括 baseURL、请求拦截器、响应拦截器等
- **api/**: 按模块组织的 API 接口定义

### 环境配置
- 开发环境 API 地址: 在 `config/app.js` 中修改 `baseURL`
- 生产环境 API 地址: 在发布前修改 `config/app.js` 中的 `baseURL`

## 功能模块

### 核心功能
- **首页**: 轮播图、商品分类、推荐商品、活动入口等
- **商品**: 商品列表、商品详情、SKU 选择、加入购物车等
- **购物车**: 购物车管理、商品结算、优惠券选择等
- **订单**: 订单创建、订单列表、订单详情、订单支付等
- **用户中心**: 个人信息、地址管理、收藏管理、优惠券管理等
- **活动**: 秒杀、拼团、砍价、积分商城等
- **客服**: 在线客服、消息中心等

### 特色功能
- **多端适配**: 一套代码适配多个平台
- **响应式设计**: 适配不同屏幕尺寸的设备
- **性能优化**: 图片懒加载、组件按需加载等
- **用户体验**: 流畅的动画效果、友好的交互界面
- **安全保障**: 数据加密传输、权限控制等

## 部署说明

### 微信小程序部署
1. 在 HBuilderX 中点击 "发行" -> "小程序-微信"
2. 填写小程序名称、AppID、版本号等信息
3. 点击 "发行" 按钮，等待构建完成
4. 在微信开发者工具中导入构建后的代码
5. 提交代码审核，审核通过后发布

### H5 部署
1. 在 HBuilderX 中点击 "发行" -> "H5"
2. 填写网站标题、域名等信息
3. 点击 "发行" 按钮，等待构建完成
4. 将构建后的文件部署到 Web 服务器
5. 配置服务器域名，确保能正常访问

### App 部署
1. 在 HBuilderX 中点击 "发行" -> "原生 App-云打包"
2. 填写应用名称、版本号、包名等信息
3. 选择 Android/iOS 平台
4. 点击 "打包" 按钮，等待打包完成
5. 下载打包后的安装包，进行测试和发布

## 开发规范

### 代码风格
- 遵循 Vue 官方风格指南
- 组件名使用 PascalCase
- 方法和变量名使用 camelCase
- 文件名使用 kebab-case

### 命名规范
- 组件名: PascalCase（如：GoodsList.vue）
- 方法名: camelCase（如：getGoodsList()）
- 变量名: camelCase（如：goodsList）
- 常量名: 全大写，单词间用下划线分隔（如：API_BASE_URL）
- 文件名: kebab-case（如：goods-detail.vue）

### 目录使用
- **api/**: 按模块组织 API 接口
- **components/**: 存放可复用组件
- **pages/**: 存放页面组件
- **libs/**: 存放工具库
- **mixins/**: 存放混合器
- **static/**: 存放静态资源

### 开发建议
1. **组件化开发**: 将常用功能封装为组件，提高代码复用率
2. **模块化开发**: 按功能模块组织代码，提高代码可维护性
3. **代码注释**: 为关键代码添加注释，提高代码可读性
4. **错误处理**: 合理处理异步操作的错误，提高应用稳定性
5. **性能优化**: 避免不必要的渲染和计算，提高应用性能

## 注意事项

### 平台差异
- **微信小程序**: 部分 API 只在微信小程序中可用，需要做平台判断
- **H5**: 部分原生能力在 H5 中不可用，需要做降级处理
- **App**: 权限申请、原生功能调用等需要适配不同平台

### API 调用
- 所有 API 调用需要通过 `api` 目录下的模块进行
- API 地址需要根据环境进行配置
- API 调用需要处理 loading 状态和错误状态

### 性能优化
- **图片优化**: 使用适当尺寸的图片，避免过大的图片
- **代码分包**: 合理使用代码分包，减少主包体积
- **按需加载**: 组件和页面按需加载，减少初始加载时间
- **缓存策略**: 合理使用缓存，减少重复请求

### 安全问题
- **数据加密**: 敏感数据需要加密传输
- **权限控制**: 合理控制用户权限，避免越权操作
- **输入验证**: 对用户输入进行验证，避免恶意输入

## 常见问题

### 1. 微信小程序登录失败
- 检查微信开发者工具是否登录
- 检查 `libs/login.js` 中的登录逻辑
- 检查 `api/user.js` 中的登录接口

### 2. API 接口调用失败
- 检查 `config/app.js` 中的 API 地址是否正确
- 检查网络连接是否正常
- 检查后端服务是否正常运行

### 3. 页面白屏或加载失败
- 检查页面路径是否正确
- 检查页面组件是否有语法错误
- 检查 API 调用是否有错误

### 4. 样式适配问题
- 使用 Flex 布局进行响应式设计
- 使用 `rpx` 单位进行尺寸适配
- 针对不同平台做样式兼容处理

### 5. 打包失败
- 检查是否有语法错误
- 检查是否有未定义的变量或方法
- 检查文件大小是否超过平台限制
